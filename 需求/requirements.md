# 英文歌曲歌词学习播放器——第一版需求文档

## 1. 文档目的与状态

- **文档目的**：作为后续架构设计、开发、测试和验收的唯一需求基线。
- **文档状态**：需求已确认，可进入架构设计阶段。
- **适用版本**：个人原型第一版。
- **产品形态**：面向 Windows 10/11 64 位系统的桌面应用。
- **变更原则**：第一版范围、关键行为或验收标准发生变化时，应先更新本文档并取得确认，再修改设计或代码。

本文档只定义产品需求和技术边界，不指定具体 Python 界面框架、数据库、打包工具或模型供应商。

## 2. 项目背景和产品目标

用户希望通过英文歌曲自然积累英文单词、短语和语境知识。传统的“逐首准备本地 MP3 和歌词”操作过于繁琐，可能降低持续使用意愿，因此第一版应以粘贴网易云公开歌单链接为主要入口，自动取得可用的曲目信息、在线音频和歌词。

第一版的核心目标是完成以下闭环：

1. 用户粘贴一个网易云公开歌单链接并导入歌单。
2. 用户从本地曲库选择歌曲并在线播放。
3. 软件按播放进度显示逐行、必要时逐词同步的歌词。
4. 用户点击歌词查看 AI 生成的简洁中文翻译和语境解析。
5. 用户可继续向 AI 追问，或选择单词、短语进行解释和收藏。
6. 受限歌曲允许关联本地 MP3/LRC 作为兜底，但本地文件不是主要导入流程。

产品成功的关键不是替代完整音乐平台，而是以尽量不中断听歌的方式帮助用户积累歌曲中的英文词句。

## 3. 目标用户与学习目标

### 3.1 目标用户

- 第一版仅供需求方个人使用，不面向公开用户发布。
- 用户英语水平大致为中等：能理解基础英文句子，但对快速演唱、俚语、隐喻和深层语境可能理解不足。
- 第一版不需要软件账号、多用户档案或跨设备同步。

### 3.2 学习目标

- 核心学习目标是积累歌曲中的英文单词、短语和表达方式。
- 软件应帮助用户理解歌词在当前歌曲中的自然中文含义、表层意思、深层语境和俚语。
- 学习操作不应默认暂停或打断歌曲；只有用户主动选择重播时才改变播放流程。
- 第一版不承担系统课程、记忆规划或考试训练功能。

## 4. 核心使用场景和用户流程

### 4.1 首次使用

1. 用户打开软件，无需登录本软件或网易云账号。
2. 用户进入设置，配置一个 OpenAI 兼容服务的地址、模型名和自己的 API 密钥。
3. API 密钥保存到 Windows 系统凭据库，不写入普通配置文件。
4. 用户粘贴网易云公开歌单链接并开始导入。
5. 软件显示歌单和曲目；歌词、在线音频地址和 AI 解析按实际使用时获取，不要求导入时全部完成。

### 4.2 日常听歌

1. 用户从侧栏打开已导入歌单。
2. 用户点击歌曲后，以该歌曲所在歌单形成播放队列，并从所点歌曲开始播放。
3. 播放页面中央显示同步歌词，右侧显示所选歌词的翻译、解析、AI 问答和收藏操作。
4. 歌曲结束后按顺序播放下一首；用户可选择顺序播放或列表循环。

### 4.3 学习歌词

1. 当前歌词行随播放进度高亮；存在逐词时间数据时，行内单词进一步随演唱进度变化。
2. 用户点击任意歌词句时，软件选择该句并显示文字解析，但音频继续播放，不跳转、不暂停、不循环。
3. 用户可点击“重播所选句”：软件从该句开始位置播放至该句结束，然后暂停。
4. 用户点击“继续”后，从所选句开头重新开始并自然播放后续歌曲。
5. 用户可拖选英文单词或连续短语，主动调用 AI 生成语境释义并加入收藏。

### 4.4 AI 深入问答

1. 用户先选择一行歌词，再在解析侧栏提出问题。
2. AI 以所选句为重点，同时参考整首歌词和已有解析回答。
3. 问答仅通过文字显示，不自动暂停、跳转或重播歌曲。
4. 问答历史按歌曲保存在本机，用户可继续追问、删除单条记录或清除该歌曲全部问答。

### 4.5 本地兜底

1. 在线歌曲因版权、地区、会员或接口原因无法播放时，仍保留在歌单并明确标记原因。
2. 用户可为该歌曲关联合法持有的本地 MP3 和对应 LRC。
3. 软件不得修改或删除用户原始文件。
4. 本地文件失效时保留歌曲和学习数据，提示用户重新选择文件或切回在线播放。

## 5. 第一版功能需求

### 5.1 导航和页面

- 采用单窗口、侧栏导航结构。
- 侧栏至少提供：歌单、正在播放、词句收藏、设置。
- 播放页面采用“歌词主区 + 解析侧栏”结构。
- 关键播放、错误和加载状态不得只使用颜色表达。
- 核心操作支持键盘完成，界面文字支持缩放。

### 5.2 曲库和歌单

- 支持粘贴网易云公开歌单链接导入。
- 单个歌单最多稳定管理 500 首歌曲。
- 保存已导入歌单的本地缓存，启动时先显示缓存，再在后台自动刷新全部已导入歌单。
- 后台刷新失败时保留上次缓存，并提供不阻断使用的提示和手动重试入口。
- 再次导入同一歌单时，不创建副本；应打开并刷新已有歌单。
- 同一歌曲出现在多个歌单时，共用同一份歌词、AI 解析、收藏和问答数据，同时保留各自歌单归属关系。
- 删除歌单时，不影响仍被其他歌单引用的歌曲。仅属于被删除歌单的数据进入待清理状态，必须再次确认后才能删除。

### 5.3 播放器

- 支持播放、暂停、上一首、下一首、进度跳转和音量控制。
- 点击歌曲后，以当前歌单为播放队列。
- 支持顺序播放和列表循环；第一版不支持随机播放。
- 支持应用内快捷键，包括空格播放/暂停和方向键相关控制。
- 支持 Windows 播放/暂停、上一首、下一首等系统媒体键。
- 窗口最小化或切换到其他软件时继续播放。
- 用户关闭窗口时，询问“退出并停止播放”或“缩小到系统托盘”。

### 5.4 用户数据管理

- 支持查看、编辑、删除歌词解析和词句收藏。
- 支持按英文或中文搜索收藏，并按来源歌曲筛选；默认按最近收藏排序。
- 支持分项清理临时缓存、AI 问答、AI 解析或曲库数据。
- 支持“一键清除全部本地数据”，但必须二次确认，并明确不会删除用户原始媒体文件。

## 6. 歌曲导入与播放需求

### 6.1 网易云公开歌单

- 输入仅接受可识别的网易云公开歌单链接。
- 软件从链接提取歌单 ID，并获取歌单名称、封面、曲目 ID、歌名、歌手、专辑、封面和时长等必要元数据。
- 无效链接、歌单不存在或无权访问时，在输入区域说明具体原因并保留原输入，便于修改后重试。
- 已导入歌单后来被删除、设为私密或暂不可访问时，保留本地缓存、歌词和学习数据，标记歌单失效，并允许重试或删除本地记录。

### 6.2 在线播放

- 点击歌曲时才获取临时音频地址，不在歌单导入阶段批量下载音频。
- 音频采用流式播放和短期缓冲，支持进度跳转所需的分段请求。
- 不提供永久下载、缓存导出、会员绕过、版权限制绕过或音质破解。
- 在线音频不可用时保留歌曲，并区分可理解的原因，例如版权、地区、会员、接口或网络限制。

### 6.3 本地文件兜底

- 允许为在线不可用歌曲关联本地 MP3 和 LRC。
- 第一版只要求 MP3 作为本地兜底音频格式。
- 本地关联关系按歌曲保存；同一歌曲在其他歌单中复用。
- 本地 MP3 或 LRC 损坏、移动或无法读取时，不删除歌曲、AI 解析、收藏或问答历史。

## 7. 歌词获取、解析和同步需求

### 7.1 歌词来源和格式

- 在线歌曲优先使用网易云返回的逐行歌词、已有翻译和逐词时间轴。
- 本地兜底允许用户关联带时间标记的 LRC。
- 歌词来源缺失或错误不得阻止歌曲进入曲库或正常播放。

### 7.2 同步显示

- 当前整行歌词应明显高亮。
- 存在逐词时间轴时，在整行高亮基础上显示行内逐词进度。
- 没有逐词时间轴时自动退回逐行高亮，不显示错误或伪造逐词进度。
- 纯文本歌词没有时间标记时，显示全文并允许 AI 解析，但禁用自动同步、时间偏移和所选句重播。

### 7.3 时间校准

- 用户可统一提前或延后整首歌词时间。
- 时间偏移按歌曲永久保存，并提供小步调整和恢复默认值的操作。
- 偏移只改变歌词显示和歌词相关播放边界，不改变音频文件本身。

### 7.4 所选句重播

- 点击歌词只选择该句并显示解析，不自动改变播放状态。
- 重播所选句时，有逐词数据则从该行开始播放至末词结束；没有逐词数据则播放至下一行开始时间。
- 缺少有效结束时间时，应禁用重播并说明原因，不使用固定时长猜测。
- 重播结束后暂停；用户点击“继续”时，从所选句开头恢复连续播放。

### 7.5 异常歌词

- 歌词损坏、缺少时间标记或无法解析时，保留歌曲并提示重新获取或关联有效歌词。
- 含少量非英语内容的歌曲保留完整歌词；AI 重点解析英文，对其他语言提供必要的中文说明。
- 用户可在之后重新获取或替换歌词，不丢失其他学习数据。

## 8. 翻译与语言学习需求

### 8.1 AI 配置

- 第一版使用 OpenAI 兼容接口。
- 设置项允许填写服务地址、模型名和 API 密钥。
- 第一版使用需求方自己的 AI 服务配置，不提供共享密钥或产品方云端代理。
- 具体服务商和基准模型在架构或测试准备阶段确定，不属于本需求文档的固定技术选择。

### 8.2 整首歌词解析

- 用户导入歌曲后，软件不得自动调用 AI。
- 用户点击“解析整首歌词”后才发起请求。
- 请求允许把整首歌词作为上下文，以获得一致的主题、代词和隐喻理解；不得发送歌曲音频。
- 每行结果采用固定结构：
  1. 自然中文翻译；
  2. 表层意思；
  3. 深层或语境含义；
  4. 确实存在时的俚语或中文语境难理解内容说明。
- 默认内容应简洁完整，每项尽量控制在一至两句话。
- AI 不应为了填满栏目而编造俚语、创作背景或文化事实。
- 结果完成后本地缓存；正常播放不得重复调用。
- 用户只有在明确选择“重新生成”时才覆盖旧结果；覆盖用户手工修改前必须提示。
- 用户可取消正在进行的解析。取消或失败时保留旧结果，不保存不完整的结构化结果。
解释俚语、连读、弱读、隐喻和文化背景
•  自动收集用户反复听错或点开的歌词短语
•  在后续听歌中自然复习，而不是强迫做题
•  针对中文用户解释中英文表达差异
•  将歌曲带来的情绪记忆转化为长期学习记录

### 8.3 逐行编辑和问答

- 用户可以逐行修改自然翻译、表层意思、深层语境和俚语说明。
- 用户修改后的内容应标记为“用户修改”。
- 用户可以围绕所选行自由追问，AI 同时参考整首歌词、所选行和已有解析。
- 追问失败不得影响已保存的解析和歌曲播放。

### 8.4 词句收藏

- 用户通过鼠标拖选一个单词或连续短语。
- 系统不得自动决定或自动收藏重点词句。
- 用户点击收藏时，单独调用 AI 生成该词句在当前歌词中的中文语境释义。
- 收藏项至少保存：英文词句、中文解释、原歌词句、来源歌曲、创建时间和个人备注。
- 收藏项支持查看、编辑、删除、搜索和按来源歌曲筛选。

### 8.5 AI 服务异常

- 断网、密钥错误、服务地址错误、模型不可用、订阅失效、限额或返回格式错误时，显示可理解的原因和重试入口。
- AI 故障不得阻止歌曲播放、英文歌词显示或已有学习数据读取。
- 软件不假设 AI 服务一定按调用量计费；首次使用时仅说明调用受用户所选服务方案、订阅或限额约束。

## 9. 数据、隐私、版权和第三方服务要求

### 9.1 本地数据

- 曲库、歌单缓存、歌词、时间偏移、AI 解析、词句收藏、问答历史和设置均保存在本机。
- API 密钥必须保存到 Windows 系统凭据库，不写入普通配置、日志、备份或 Git。
- 问答和歌词内容只在用户主动调用 AI 时发送给所配置的服务。
- 歌曲音频不得发送给 AI 服务。

### 9.2 备份和恢复

- 支持将歌单信息、歌词、解析、收藏、问答历史和设置导出为单个备份文件。
- 备份不包含 API 密钥、网易云 Cookie、在线音频临时缓存或用户原始媒体文件。
- 恢复前必须验证备份格式、完整性和版本兼容性。
- 验证失败时不得修改现有数据，并提供可理解的原因和诊断信息。
- 数据格式升级前自动创建带版本号的备份；回退旧版时允许恢复与旧版匹配的备份。

### 9.3 版权边界

- 软件仅用于个人学习和用户合法取得内容的播放辅助。
- 软件不是网易云官方客户端，也不得宣传为官方或获得官方授权。
- 不提供抓取、破解、去除保护、绕过会员、永久下载或重新分发音乐和完整歌词的能力。
- 用户关联本地 MP3/LRC 时，应确认自己有权用于个人学习。
- 测试素材优先使用原创、可公开用于项目测试的内容。

### 9.4 第三方接口

- 网易云接口和 AI 服务均属于外部依赖，可能变更、限流、失效或修改服务条款。
- 软件应隔离外部接口错误，避免第三方故障损坏本地数据。
- 是否允许发布、商业运营或扩大用户范围，必须在未来重新进行版权、隐私和服务条款评估。

## 10. 非功能需求

### 10.1 平台与硬件

- 支持 Windows 10/11 64 位。
- 最低流畅目标：双核或更高处理器、8GB 内存的普通 Windows 电脑。
- 提供普通 Windows 安装包；最终用户不应被要求手工安装 Python 或 Node.js 运行环境。
- 2026-07-17 确认首个可交付版本接受 `PARTIAL_OFFLINE`：优先完整交付本地 MP3/LRC 播放、歌词学习和 AI 能力；网易云在线验证未通过时保持入口禁用并说明原因，不以等待非官方接口阻塞首版。
- 首版以精简和尽快可用为优先；未启用的网易云在线能力及其专用 Node.js 运行时不进入安装包，后续验证门通过后再作为受测组件加入。

### 10.2 性能

- 在最低目标电脑上，从启动到显示缓存曲库不超过 5 秒。
- 普通稳定网络下，首次导入包含 100 首歌曲的公开歌单，在 30 秒内显示可浏览曲目列表。
- 普通稳定网络下，点击可播放歌曲后在 5 秒内开始出声。
- 正常播放并同步歌词时，应用总内存占用不高于 700MB。
- 对歌词源时间轴本身正确的内容，高亮与实际播放时间误差不超过 500 毫秒。

### 10.3 稳定性

- 连续顺序播放测试歌单 4 小时，期间正常切歌、同步歌词，不得崩溃或要求重启。
- 临时网络波动、AI 请求失败或歌单刷新失败不得造成应用崩溃或本地数据损坏。
- 后台任务不得阻塞歌曲播放和主要界面操作。

### 10.4 可用性与可访问性

- 核心播放和导航操作支持键盘。
- 支持文字缩放。
- 加载、选中、不可用和错误状态不得只依赖颜色。
- 面向非技术用户的错误信息应说明发生了什么、是否影响已有数据以及下一步可执行操作。

## 11. 异常情况与恢复方式

| 异常情况 | 预期行为 |
| --- | --- |
| 网易云歌单链接格式无效 | 原地说明格式问题，保留输入，允许修改重试 |
| 歌单不存在或不可访问 | 说明不存在、私密或接口异常；不创建空歌单 |
| 已导入歌单变私密或删除 | 保留缓存和学习数据，标记失效，允许重试或删除本地记录 |
| 重复导入同一歌单 | 打开已有歌单并刷新，不创建副本 |
| 在线歌曲受版权、地区或会员限制 | 保留歌曲并标记原因，允许关联本地 MP3/LRC |
| 播放途中网络中断 | 显示缓冲状态并有限次数自动重试；仍失败则暂停在当前位置 |
| 歌词缺失 | 保留并允许播放，允许之后重新获取或补充歌词 |
| 歌词损坏或无有效时间轴 | 保留歌曲；显示可用文本，禁用依赖时间轴的功能并提示修复 |
| 本地 MP3/LRC 失效 | 保留关联和学习数据，提示重新选择或切回在线来源 |
| AI 密钥、订阅、模型或网络异常 | 不影响播放和已有数据，显示原因并允许修复后重试 |
| AI 返回格式不完整 | 不覆盖旧解析；说明结果无效并允许重试 |
| AI 任务被取消 | 中止当前任务并保留旧解析 |
| 磁盘空间不足 | 停止当前写入，保护已有数据，说明所需空间并提供缓存清理入口 |
| 备份损坏或不兼容 | 验证失败后停止，不修改现有数据，提供诊断信息 |
| 文件重名或重复歌曲 | 以稳定歌曲标识和来源关系去重，不覆盖无关数据 |

## 12. 技术约束

- 主应用业务和桌面功能使用 Python 开发。

- 应将网易云接口、AI 接口、播放、歌词、数据存储和界面逻辑保持清晰边界，便于替换外部服务或定位故障。

### 12.1 Git 与版本管理

- 正式开发开始时初始化本地 Git 仓库，并同步到私有 GitHub 仓库。
- 采用“主分支 + 短期功能分支”方式开发；主分支始终保持可运行。
- 每个完整、可独立验证的功能形成单独提交；不得把多个无关功能混入同一提交。
- 功能分支通过对应测试后才合并到主分支。
- 正式版本使用类似 `v0.1.0` 的版本标签，并维护变更日志，记录新增、修复和已知问题。
- 发布标签前必须通过自动测试和完整冒烟测试。
- API 密钥、Cookie、用户歌曲、用户歌词、备份、缓存和问答历史不得提交 Git。

## 14. 验收标准

### 14.1 固定测试材料

- 在线主流程使用公开网易云歌单：`https://music.163.com/playlist?id=5243040566`。
- 验收开始时记录该歌单是否仍可公开访问、实际曲目数和歌词覆盖情况。
- 项目制作一组无版权争议的原创测试素材：简单旋律加原创英文演唱的 MP3，以及对应 LRC。
- 原创素材至少覆盖：普通逐行时间轴、可验证的逐词时间轴、少量非英语内容、整体时间偏移和所选句重播边界。
- 若在线歌单缺少上述歌词类型，使用原创素材补足，不更换已经确认的核心测试目标。

### 14.2 功能验收

1. 粘贴固定歌单链接后，能导入并显示歌单和可获得的曲目元数据。
2. 100 首规模的歌单在普通稳定网络下 30 秒内出现可浏览列表。
3. 重复导入同一链接不会创建副本。
4. 点击可播放歌曲后 5 秒内出声，并能播放、暂停、调整进度和音量、切换歌曲。
5. 播放队列来自当前歌单，顺序播放和列表循环行为正确。
6. 最小化后继续播放，系统媒体键能够控制播放器。
7. 有效逐行歌词正确高亮；有逐词数据时行内进度正确，无逐词数据时正常退回逐行。
8. 歌词源时间轴正确时，同步误差不超过 500 毫秒。
9. 点击歌词只显示解析，不改变当前播放进度。
10. 重播所选句按确认的时间边界播放一次并暂停；点击继续后从该句开头连续播放。
11. 用户可手动调整并永久保存歌词整体偏移，也可恢复默认值。
12. 纯文本歌词能够显示和 AI 解析，同时正确禁用同步、偏移和重播。
13. 用户主动点击后能生成整首结构化 AI 解析，播放过程不受阻断。
14. AI 解析可逐行编辑并标记用户修改，重新生成前提示覆盖风险。
15. 用户可围绕所选句追问，问答历史能够保存、恢复和删除。
16. 用户可拖选词句、生成语境释义、收藏、编辑、搜索、筛选和删除。
17. 受限在线歌曲可关联原创本地 MP3/LRC，并完成播放、同步和重播。
18. 备份能够导出；在干净测试数据环境中恢复后，歌单、歌词、解析、收藏、问答和设置与备份前一致。
19. 升级测试前自动生成版本备份，旧版回退时能够恢复匹配备份。
20. 无效链接、断网、歌词缺失、AI 失败、磁盘不足和损坏备份均按第 11 节处理，且不损坏已有数据。

### 14.3 AI 质量验收

- 使用架构阶段记录的基准模型和同一测试配置。
- 从测试歌曲中随机抽查至少 20 行。
- 至少 80% 的抽查行满足：自然翻译基本准确、表层意思清晰、深层语境合理。
- 不得出现明显编造的俚语、文化背景或歌曲事实。
- 不存在俚语的行允许明确写“无特别俚语”，不得为了结构完整而虚构内容。

### 14.4 性能和稳定性验收

- 最低目标电脑上 5 秒内显示缓存曲库。
- 正常播放和歌词同步时总内存不高于 700MB。
- 连续播放测试歌单 4 小时无崩溃、无必须重启、无明显持续内存增长导致的失稳。
- 后台歌单刷新和 AI 解析期间，播放和主要界面操作保持可用。

### 14.5 发布验收

- 自动单元测试和集成测试全部通过。
- 手工冒烟测试覆盖：安装、启动、导入、播放、歌词、句子重播、AI、收藏、备份恢复和异常处理。
- 主分支处于可运行状态。
- 发布版本具有版本标签和变更日志。
- 仓库中不存在 API 密钥、Cookie、用户媒体、用户歌词、备份或运行缓存。
