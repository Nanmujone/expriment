# 导航与页面模块任务清单

> 直接采用测试驱动开发（TDD）：每项任务先写能表达行为的失败测试，再写最小实现使测试通过，最后重构；只有对应测试和“完成定义”均通过后才能勾选。

**目标：** 提供单窗口、侧栏导航、播放学习页面、快捷键、可访问性和托盘交互。

**依赖：** 各领域模块的应用服务接口与只读 ViewModel；播放器状态接口。

- [x] `NAV-02` 实现 `MainWindowShell` 的单窗口布局与侧栏占位区；测试窗口始终只有一个主内容区。

- [ ] 四个主入口、播放学习页、快捷键、缩放和托盘流程全部可用。
