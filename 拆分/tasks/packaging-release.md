# 打包与发布模块任务清单

> 直接采用测试驱动开发（TDD）处理可自动验证的打包脚本和发布检查；安装、升级、卸载及 Windows 兼容性必须另有真实环境证据，未验证不得勾选。

**目标：** 生成普通 Windows 用户无需安装 Python 或 Node.js 即可使用的可复现安装包，并建立安全的安装、升级、卸载和 Git 发布流程。

**依赖：** `application-integration` 及其依赖的全部模块。

## 最小任务

- [ ] `PKG-01` 锁定并记录 Python、PySide6/Qt、Node.js LTS、Node 依赖、PyInstaller 和安装器工具版本；验证可从干净环境复现依赖。
- [ ] `PKG-02` 创建 PyInstaller spec 和构建脚本，包含 Python 主程序、迁移资源、许可证和启动元数据；构建过程不得读取或嵌入开发机凭据与用户数据。
- [ ] `PKG-03` 打包 Qt 运行库、平台插件、Qt Multimedia 后端和所需编解码组件；在无开发环境的 Windows 10/11 虚拟机验证启动与原创 MP3 播放。
- [ ] `PKG-04` 随包携带固定版本 Node.js LTS 运行时、锁定依赖和网易云适配层；验证随机回环端口、标准输入会话令牌、健康检查、有限重启和受控退出。
- [ ] `PKG-05` 定义安装目录和 Windows 用户数据目录；验证运行时数据库、日志、缓存、备份和设置不写入安装目录。
- [ ] `PKG-06` 生成普通 Windows 安装器和卸载器，包含开始菜单/桌面入口、版本信息、许可证与卸载项；最终用户无需手工安装 Python 或 Node.js。
- [ ] `PKG-07` 测试全新安装、重复安装和修复安装；启动后完成缓存读取、本地原创素材播放、Qt 插件和 Node 健康检查烟雾测试。
- [ ] `PKG-08` 测试跨 Schema 版本升级：先创建并验证带版本号备份，再执行迁移；备份失败、空间不足或迁移失败时阻止升级并保持旧数据可读。
- [ ] `PKG-09` 测试回退旧版与匹配备份恢复；不兼容备份必须在修改现有数据前拒绝并给出诊断。
- [ ] `PKG-10` 测试卸载默认保留用户数据，并提供删除应用数据的明确二次确认；无论选择何项都不得修改或删除外部 MP3/LRC。
- [ ] `PKG-11` 建立安装包烟雾矩阵：安装、启动、导入、播放、歌词、重播、AI、收藏、备份恢复、磁盘不足、断网和 Node 崩溃；真实外部服务不可用项明确标记未验证。
- [ ] `PKG-12` 在打包前后扫描源码、配置、构建日志、产物和 Git 历史，阻止 API 密钥、Cookie、用户媒体/歌词、问答、数据库、备份、缓存、日志、诊断包和临时音频地址进入发布物或仓库。
- [ ] `PKG-13` 为 Python、Node、Qt 和随包二进制生成 SPDX 或 CycloneDX SBOM，附第三方许可证与来源清单；高危漏洞或许可证风险必须阻止发布或形成明确批准记录。
- [ ] `PKG-14` 为安装器和主要可执行文件生成 SHA-256 校验值；如配置代码签名则验证签名和时间戳，未签名时不得宣称已签名。
- [ ] `PKG-15` 维护 `CHANGELOG.md`，按新增、修复、已知问题和迁移/兼容说明记录发布内容；版本号与应用、安装器、备份清单保持一致。
- [ ] `PKG-16` 仅在自动测试、固定材料验收、安装烟雾、敏感扫描和 SBOM 门全部通过后创建形如 `v0.1.0` 的 Git 标签；不得移动或复用已发布标签。
- [ ] `PKG-17` 归档构建命令、锁文件哈希、测试环境、测试结果、SBOM、校验值、变更日志和发布阻塞项，形成可复核发布记录。

## 完成定义

- [ ] 干净 Windows 10/11 电脑无需 Python/Node.js 开发环境即可安装、启动、播放原创素材并卸载。
- [ ] 安装、升级、回退和卸载不会损坏现有数据或外部 MP3/LRC，失败路径均可恢复。
- [ ] 标签、变更日志、敏感扫描、SBOM、产物校验和发布证据齐全且相互对应。
