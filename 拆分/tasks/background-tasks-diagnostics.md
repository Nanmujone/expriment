# 后台任务与诊断模块任务清单

**目标：** 统一管理可取消任务、并发、重试、用户错误和脱敏诊断。

**依赖：** 无；其他异步模块应复用本模块。

## 最小任务

- [ ] `TASK-01` 定义任务 ID、类型、状态、进度、取消能力、结果和错误 DTO。
- [ ] `TASK-02` 实现任务注册表及合法状态转换；测试排队、运行、成功、失败和取消。
- [ ] `TASK-03` 实现取消令牌从应用服务传至适配器，并丢弃取消后的迟到结果。
- [ ] `TASK-04` 实现资源并发限制：歌单刷新小并发、同歌曲同类 AI 任务互斥、受控文件工作线程。
- [ ] `TASK-05` 实现连接超时、总超时、有限重试、指数退避和随机抖动。
- [ ] `TASK-06` 建立可重试判定：只自动重试幂等读取和短暂网络错误。
- [ ] `TASK-07` 明确认证、权限、版权、会员、格式、磁盘不足和备份不兼容不自动重试。
- [ ] `TASK-08` 实现用户错误模型：“发生了什么、是否影响已有数据、下一步操作”。
- [ ] `TASK-09` 实现轮转日志；默认不记录密钥、Cookie、歌词/问答全文和媒体路径。
- [ ] `TASK-10` 实现诊断包清单及生成器；用户确认前显示将包含和排除的内容。
- [ ] `TASK-11` 实现关闭协调：停止接收新任务、取消可取消任务、等待必要写入、报告未完成任务。
- [ ] `TASK-12` 验证后台错误通知不暂停播放、不抢焦点、不阻塞 UI 线程。
- [ ] `TASK-13` 运行 `pytest tests/tasks tests/diagnostics -q`，包括并发、超时和脱敏测试。
- [ ] `TASK-14` 对任务状态、诊断包和错误提示执行固定敏感数据扫描；覆盖 API 密钥、Cookie、完整歌词/问答、临时音频地址和本地媒体路径，发现命中即使测试失败。

## 完成定义

- [ ] 所有后台任务都可查询状态，允许时可取消，错误具有明确下一步。
- [ ] 重试次数有上限，取消后的迟到结果不会写库。
- [ ] UI 线程无阻塞网络、备份校验或大文件操作。
