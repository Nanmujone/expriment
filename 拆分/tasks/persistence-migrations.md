# 持久化与迁移模块任务清单

**目标：** 建立 SQLite 数据模型、仓储、短事务、查询索引和安全迁移基础。

**依赖：** 无；这是多数业务模块的前置模块。

## 最小任务

- [x] `DB-01` 建立 `src/english_player/infrastructure/persistence/`、SQLAlchemy 2 基础配置和测试数据库夹具。
- [x] `DB-02` 将设计实体映射为物理表：歌单、歌曲、关系、歌词、解析版本、问答、收藏、本地媒体关联、设置引用、任务和待清理项。
- [x] `DB-03` 为稳定远端 ID、歌单歌曲关系和版本关系添加外键与唯一约束，并编写约束失败测试。
- [x] `DB-04` 为启动曲库、收藏搜索/排序、待清理扫描和问答查询建立索引并检查查询计划。
- [x] `DB-05` 实现 `UnitOfWork`：短事务、显式提交、异常回滚和会话关闭。
- [ ] `DB-06` 为各领域已定义的仓储端口提供 SQLAlchemy 适配器；本模块不得替领域定义端口，适配器不得向上泄露 ORM 对象。
- [x] `DB-07` 实现单写入协调策略；模拟 SQLite 锁定并返回可重试的标准错误。
- [x] `DB-08` 实现事务外验证、事务内批量写入模式；测试网络/文件/AI 调用不会发生在事务中。
- [x] `DB-09` 初始化 Alembic 并生成首版迁移；对空数据库执行升级并验证全部约束和索引。
- [x] `DB-10` 定义迁移预检与 dry-run 结果，并让迁移执行器接受应用层注入的“备份已验证”凭据；没有有效凭据、备份失败或空间不足时必须阻止迁移，本模块不得反向依赖用户数据模块。
- [x] `DB-11` 编写迁移失败回滚和重新启动恢复测试；原数据库必须保持可用。
- [x] `DB-12` 配置用户数据目录，禁止向安装目录写入数据库和缓存。
- [x] `DB-13` 运行 `alembic upgrade head`、`pytest tests/persistence tests/migrations -q`。
- [x] `DB-14` 对数据库新增、更新、批量写入和迁移分别注入磁盘不足故障；验证事务回滚、旧数据可读、说明所需空间并返回缓存清理入口，且不自动重试。

## 完成定义

- [ ] 数据约束、索引、事务回滚、锁定和磁盘异常均有测试。
- [ ] 仓储只返回领域实体或 DTO，不向上泄露 SQLAlchemy 模型。
- [ ] 数据升级前有可验证备份，失败不会损坏现有数据。
