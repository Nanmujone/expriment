# AI 学习模块任务清单

**目标：** 在用户主动触发时生成、校验、版本化保存歌词解析和按歌曲组织的问答。

**依赖：** `lyrics`、`external-services`、`persistence-migrations`、`background-tasks-diagnostics`。

## 最小任务

- [ ] `AI-01` 定义服务地址、模型名和非敏感生成参数配置；API 密钥仅保存凭据引用。
- [ ] `AI-02` 定义固定提示词版本、逐行稳定 ID 和解析结果 JSON Schema。
- [ ] `AI-03` 实现能力探测：优先 JSON Schema，不支持时切换严格 JSON 路径。
- [ ] `AI-04` 实现 AI 请求上下文构建器；只包含必要歌词、所选行、已有解析和问答历史。
- [ ] `AI-05` 实现整首解析请求；只有用户点击后才调用，不在导入或播放时自动调用。
- [ ] `AI-06` 实现本地 Schema 与业务校验；校验行数、行 ID、四类字段和完整性。
- [ ] `AI-07` 实现原子保存完整解析；失败、取消或迟到响应不得保存部分结果或覆盖旧版本。
- [ ] `AI-08` 实现“重新生成”新版本；检测用户修改并在提交请求前要求再次确认。
- [ ] `AI-09` 实现逐行编辑及“用户修改”标记；其他字段和版本保持不变。
- [ ] `AI-10` 实现按歌曲问答的新增、继续追问、删除单条和清空全部记录。
- [ ] `AI-11` 实现词句语境释义用例，输入必须来自单行内已校验的连续选择。
- [ ] `AI-12` 将密钥、地址、模型、订阅、限额、网络和格式错误映射为可操作错误。
- [ ] `AI-13` 实现取消信号贯穿应用服务与适配器；测试取消后的迟到响应被丢弃。
- [ ] `AI-14` 测试诊断和日志中不出现密钥、完整问答或不必要歌词全文。
- [ ] `AI-15` 用固定模型 `gpt-5.6-terra`、固定提示词/参数和固定测试歌曲随机抽查至少 20 行；记录测试日期与逐行判定，要求至少 80% 同时满足自然翻译基本准确、表层意思清晰、深层语境合理，且不得编造俚语、文化背景或歌曲事实。
- [ ] `AI-16` 运行 `pytest tests/ai_learning -q`。
- [ ] `AI-17` 由 AI 学习领域定义 `AIProvider`、`CredentialProvider` 和解析/问答仓储端口及行为契约；OpenAI、Windows 凭据和 SQLAlchemy 组件只实现端口。

## 完成定义

- [ ] AI 只能由用户主动触发，故障不影响播放和旧结果。
- [ ] 保存前同时通过结构与业务校验。
- [ ] 密钥、音频、媒体路径和无关歌曲数据不会发送给 AI。
