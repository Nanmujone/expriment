# 曲库与歌单模块任务清单

> 每个任务以领域测试或应用服务测试通过为完成条件。

**目标：** 安全导入、缓存、刷新和删除最多 500 首歌曲的网易云公开歌单。

**依赖：** `persistence-migrations`、`external-services`、`background-tasks-diagnostics`。

## 最小任务

- [ ] `LIB-01` 定义稳定的 `PlaylistId`、`SongId`、歌单快照和歌单歌曲关系 DTO；测试相等性和非法值。
- [ ] `LIB-02` 实现公开歌单链接规范化与 ID 提取；覆盖标准链接、分享链接、无效域名、缺失 ID 和保留原输入。
- [ ] `LIB-03` 定义 `PlaylistProvider` 端口及标准错误类型；用假适配器完成契约测试。
- [ ] `LIB-04` 实现远端歌单快照标准化，限制最多 500 首并按远端歌曲 ID 去重；测试顺序保持稳定。
- [ ] `LIB-05` 实现 `ImportPlaylist(link)`；重复导入时返回已有歌单并进入刷新，不创建副本。
- [ ] `LIB-06` 实现远端快照差异计算；测试新增、更新、移除关系不会直接删除歌曲学习数据。
- [ ] `LIB-07` 实现 `RefreshPlaylist(playlist_id)` 的“事务外获取和验证、单事务写入”；测试失败保留旧缓存。
- [ ] `LIB-08` 实现 `RefreshAllPlaylists()` 的后台调度和逐歌单结果汇总；一个歌单失败不得阻断其余歌单。
- [ ] `LIB-09` 实现启动时优先读取缓存的查询服务；测试不依赖网络即可在 5 秒目标内返回可展示数据。
- [ ] `LIB-10` 实现歌单失效状态、分类原因、手动重试和本地记录删除入口。
- [ ] `LIB-11` 实现 `DeletePlaylist`：仅删除关系，共享歌曲保留，孤立歌曲进入 30 天待清理。
- [ ] `LIB-12` 实现重新导入或重新引用时撤销待清理标记；覆盖临界日期和并发写入。
- [ ] `LIB-13` 运行 `pytest tests/library -q` 和对应数据库集成测试。
- [ ] `LIB-14` 由曲库领域定义 `PlaylistRepository` 端口及行为契约；领域测试使用内存假实现，SQLAlchemy 实现留给持久化适配器。

## 完成定义

- [ ] 导入、重复导入、缓存优先、后台刷新、失效重试和删除恢复均有测试。
- [ ] 网络调用不在数据库事务中执行。
- [ ] 任何刷新失败均不会破坏上次可用缓存或学习数据。
