# 应用集成与 UI 接线模块任务清单

> 直接采用测试驱动开发（TDD）：先用应用服务测试或 `pytest-qt` 写出失败行为，再实现最小接线并重构；只有测试和人工验收均有证据后才能勾选。

**目标：** 通过应用服务和 ViewModel 把全部核心用户动作接入领域能力，统一确认、错误、取消、隐私提示和跨模块编排，禁止 UI 直接访问数据库、HTTP、文件系统或底层播放对象。

**依赖：** `navigation-pages`、`playlist-library`、`player`、`lyrics`、`ai-learning`、`phrase-favorites`、`user-data`、`persistence-migrations`、`external-services`、`background-tasks-diagnostics`。

## 最小任务

- [ ] `APP-01` 建立应用组合根和依赖注入：领域层拥有端口，持久化、外部服务、Qt 和文件系统适配器实现端口；用架构测试阻止领域层依赖适配器或 UI。
- [ ] `APP-02` 接线启动流程：初始化日志和数据目录、执行数据库预检、在需要迁移时先取得已验证升级备份凭据、显示缓存曲库、启动 Node 健康检查并调度后台刷新；网络和 Node 不得阻塞缓存展示。
- [ ] `APP-03` 接线公开歌单链接的导入、重复导入、后台刷新、失败重试、失效状态和本地记录删除；保留无效输入，所有错误均显示影响和下一步。
- [ ] `APP-04` 接线歌曲点击、队列构建、播放/暂停、进度、音量、上一首/下一首、循环模式、快捷键、媒体键和托盘；测试所有入口汇聚到同一播放协调器。
- [ ] `APP-05` 接线受限歌曲关联本地 MP3/LRC；选择前说明文件仅只读引用且要求用户确认有权用于个人学习，取消时不保存关联。
- [ ] `APP-06` 接线歌词获取/替换、纯文本/逐行/逐词降级、同步高亮、永久偏移与恢复默认、选句重播和继续；普通选中歌词不得改变播放状态。
- [ ] `APP-07` 接线 AI 地址、模型、参数和密钥设置；密钥只经凭据端口写入 Windows Credential Manager，界面、普通配置、日志和数据库不得回显或保存密钥值。
- [ ] `APP-08` 接线整首 AI 解析、取消、逐行编辑、重新生成和旧版本保留；首次发送前明确说明必要歌词/上下文会发往用户配置的服务，用户修改被覆盖前再次确认。
- [ ] `APP-09` 接线按歌曲问答的新增、继续、删除单条和清空全部；发送前展示上下文范围，删除操作使用与预览内容绑定且会过期的确认令牌。
- [ ] `APP-10` 接线单行连续词句选择、AI 语境释义、收藏、搜索、来源筛选、排序、备注编辑和删除；取消或 AI 失败时不得创建残缺收藏。
- [ ] `APP-11` 接线分项清理、全量清理、待清理查看和 30 天内恢复；所有删除先显示数量、影响范围和外部媒体保护说明。
- [ ] `APP-12` 接线 30 天到期后的再次确认删除；到期事件只能把状态改为“等待确认”，用户确认前不得物理删除，确认后仍需重算引用并保护共享歌曲。
- [ ] `APP-13` 接线备份导出、验证和恢复；导出前提示备份未加密、包含的个人学习数据且不会自动上传，恢复前显示校验/兼容结果，任何失败不修改现有数据。
- [ ] `APP-14` 接线诊断包生成；确认前显示包含项和排除项，测试取消时不读取或打包敏感内容。
- [ ] `APP-15` 接线关闭流程：退出/托盘选择、停止接收新任务、取消可取消任务、等待必要写入、保存设置、停止播放、受控关闭 Node、数据库和日志。
- [ ] `APP-16` 建立逐条追踪矩阵：将需求第 5～12 节每条第一版要求和第 14.2 节 20 条功能验收逐项映射到应用用例、UI 入口、自动测试、人工证据和异常路径；缺少任一列时不得标记完成。
- [ ] `APP-17` 固化测试材料：记录公开歌单 `https://music.163.com/playlist?id=5243040566` 的测试日期、可访问性、曲目数和歌词覆盖；只提交原创/可再分发的 MP3/LRC 夹具，覆盖逐行、逐词、少量非英语、整体偏移、纯文本和句尾边界。
- [ ] `APP-18` 在 Windows 10/11 最低目标电脑记录性能：缓存曲库 ≤5 秒、100 首歌单可浏览 ≤30 秒、点击可播放歌曲首声 ≤5 秒、歌词误差 ≤500ms、正常播放同步总内存 ≤700MB；记录硬件、网络、构建版本和测量方法。
- [ ] `APP-19` 执行 4 小时顺序播放和内存趋势测试，并在后台刷新与 AI 请求期间验证播放和主要 UI 可用；真实环境未完成时保持未勾选，禁止用模拟结果代替。
- [ ] `APP-20` 执行跨模块磁盘不足场景：设置、歌词/AI/收藏保存、数据库迁移、备份导出/恢复分别失败时均保留旧数据、给出所需空间和缓存清理入口，不进入无限重试。
- [ ] `APP-21` 运行 `pytest tests/application tests/integration tests/ui -q` 和架构依赖测试，保存固定材料、隐私确认、取消、故障恢复和性能证据。

## 完成定义

- [ ] 所有核心用户动作均通过应用服务接线，UI 不直接访问数据库、外部 HTTP、文件系统或底层播放对象。
- [ ] 发送、导出、关联媒体和删除等隐私/破坏性动作都有范围明确、可取消、可测试的确认流程。
- [ ] 需求逐条追踪无缺口，固定材料、性能、稳定性、备份往返和磁盘不足均有可复核证据。
