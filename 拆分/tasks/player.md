# 播放器模块任务清单

**目标：** 提供在线/本地音频播放、队列控制、系统媒体集成和句子重播状态机。

**依赖：** `playlist-library`、`lyrics`、`external-services`、`background-tasks-diagnostics`。

## 最小任务

- [ ] `PLY-01` 定义播放队列、当前索引、播放模式、音量和播放状态值对象；测试空队列及边界索引。
- [ ] `PLY-02` 实现从当前歌单和所点歌曲构建队列；测试歌曲顺序与起始索引。
- [ ] `PLY-03` 实现顺序播放和列表循环的上一首/下一首规则；明确第一首、末首和单曲队列行为。
- [ ] `PLY-04` 由播放器领域定义 `AudioSourceProvider`、`MediaFileProvider`、`PlaybackEngine` 端口和统一音频来源 DTO；用假适配器完成行为测试，具体 Qt、在线和文件实现留给适配器。
- [ ] `PLY-05` 实现来源解析：验证本地关联、按需请求在线临时地址、允许用户显式切换；临时地址不得入库。
- [ ] `PLY-06` 将版权、地区、会员、接口、网络和本地文件异常映射为稳定错误分类及下一步操作。
- [ ] `PLY-07` 封装 Qt Multimedia 为 `PlaybackEngine`，支持加载、播放、暂停、停止、音量和进度跳转；使用测试替身验证命令映射。
- [ ] `PLY-08` 实现 `PlaybackCoordinator` 状态机和单一事件出口；测试非法状态转换被拒绝。
- [ ] `PLY-09` 实现缓冲、超时、有限重试及播放中断后停留当前位置；后台刷新不得抢占播放请求。
- [ ] `PLY-10` 实现播放结束自动切歌；覆盖顺序结束、列表循环和错误歌曲跳过策略。
- [ ] `PLY-11` 实现应用快捷键、Windows 媒体键和托盘命令到协调器的统一映射。
- [ ] `PLY-12` 实现 `SegmentReplay`：只接受歌词模块给出的有效毫秒起止边界，到达句尾后暂停。
- [ ] `PLY-13` 实现“继续”：从所选句开头恢复自然连续播放；普通选句不改变播放状态。
- [ ] `PLY-14` 使用原创 MP3 测试本地播放、跳转、切歌和句子重播；记录在线 Range/跳转验证结果。
- [ ] `PLY-15` 运行 `pytest tests/player -q`，并执行 4 小时稳定性测试脚本。

## 完成定义

- [ ] 基础控制、两种队列模式、后台播放、媒体键和托盘控制可用。
- [ ] 无有效句尾时明确禁用重播，不使用固定时长猜测。
- [ ] 在线临时地址和音频缓存不可永久保存或导出。
