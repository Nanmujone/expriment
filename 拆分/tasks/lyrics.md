# 歌词模块任务清单

**目标：** 标准化歌词、同步当前行/词、保存偏移并提供可靠的句子重播边界。

**依赖：** `external-services`、`persistence-migrations`。

## 最小任务

- [x] `LYR-01` 定义纯文本、逐行、逐词三种歌词能力和毫秒整数时间模型。
- [ ] `LYR-02` 实现在线歌词 DTO 到内部模型的标准化；保留原文、翻译和少量非英语内容。
- [x] `LYR-03` 实现 LRC 解析器；覆盖多时间标签、偏移标签、空行、损坏行和编码异常。
- [ ] `LYR-04` 实现逐词时间轴校验；无效逐词数据自动降级到逐行，不伪造进度。
- [x] `LYR-05` 实现纯文本降级；仍允许全文显示和 AI 解析，但禁用同步、偏移和重播。
- [x] `LYR-06` 实现按播放毫秒定位当前行的增量算法；测试边界、跳转和倒退。
- [ ] `LYR-07` 实现当前行内逐词定位；测试首词前、词间、末词后和重复时间戳。
- [ ] `LYR-08` 实现每首歌曲永久偏移的调整、读取和恢复默认；偏移不得修改原始时间戳。
- [ ] `LYR-09` 实现逐词歌词句尾计算：使用末词结束时间，缺失时返回不可重播原因。
- [x] `LYR-10` 实现逐行歌词句尾计算：使用下一行开始时间，最后一行缺少边界时禁用重播。
- [ ] `LYR-11` 实现歌词重新获取/替换；保留歌曲的 AI、收藏、问答等其他学习数据。
- [ ] `LYR-12` 当前 AI 输出仅包含标准化歌词全文且不含媒体路径；稳定行 ID 和语言提示仍待接入。
- [ ] `LYR-13` 用原创纯文本、逐行 LRC、逐词样本和损坏样本运行 `pytest tests/lyrics -q`。
- [ ] `LYR-14` 已定义 `LyricsProvider`、`LyricsRepository` 端口；完整行为契约与 SQLAlchemy 适配器仍待实现。

## 完成定义

- [ ] 三种能力级别严格按矩阵启用或禁用功能。
- [ ] 歌词同步误差具备不超过 500ms 的验收测试。
- [ ] 损坏或缺失歌词不阻止歌曲入库和播放。
