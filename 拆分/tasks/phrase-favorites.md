# 词句收藏模块任务清单

**目标：** 收藏用户主动选择的歌词单词或连续短语，并支持本地管理和检索。

**依赖：** `ai-learning`、`persistence-migrations`、`lyrics`。

## 最小任务

- [ ] `FAV-01` 定义收藏实体：英文词句、中文解释、原歌词快照、稳定歌曲 ID、创建时间和备注。
- [ ] `FAV-02` 实现单行歌词内选择范围校验；拒绝跨行、空白和非连续范围，不自动扩展选择。
- [ ] `FAV-03` 实现收藏预览；在用户点击收藏前不得调用 AI 或写库。
- [ ] `FAV-04` 调用 AI 词句释义用例，并处理取消、失败和格式错误；失败时不创建残缺收藏。
- [ ] `FAV-05` 原子保存收藏及歌词文本快照；歌词以后替换不得改变该快照。
- [ ] `FAV-06` 实现收藏详情查询与个人备注编辑。
- [ ] `FAV-07` 实现删除收藏并要求明确确认；删除不影响歌曲和歌词。
- [ ] `FAV-08` 实现英文/中文搜索，测试大小写、前后空格和无结果。
- [ ] `FAV-09` 实现按来源歌曲筛选和最近收藏排序；为查询建立对应索引。
- [ ] `FAV-10` 验证同一词句在不同歌曲或不同上下文中可分别收藏。
- [ ] `FAV-11` 运行 `pytest tests/favorites -q`。
- [ ] `FAV-12` 由收藏领域定义 `FavoriteRepository` 和语境释义用例端口；领域测试使用假实现，持久化和 AI 模块只提供适配器。

## 完成定义

- [ ] 只有用户明确选择并确认后才调用 AI 和创建收藏。
- [ ] 收藏可查看、编辑、删除、搜索、筛选和排序。
- [ ] 收藏上下文不因后续歌词替换而丢失。
